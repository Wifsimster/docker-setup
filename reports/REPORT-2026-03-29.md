# GitHub Actions Runners — Migration Report

**Date:** 2026-03-29
**Scope:** Mutualization of 6 self-hosted GitHub Actions runners

---

## Summary

6 separate runner installations were consolidated into a single set of shared binaries using hardlinks, recovering **7.3 Go** of disk space on a system that was at **100% capacity**.

| Metric | Before | After |
|--------|--------|-------|
| Disk available | 0 Mo (100%) | 7.3 Go (93%) |
| Total runner size | ~11.8 Go | ~4.2 Go |
| Binary copies | 6 | 1 + 5 hardlinks |
| Runners online | 6 | 6 |

---

## Root Cause

Each runner maintained:
- **2 versions** of binaries (`bin.2.332.0` + `bin.2.333.0`, ~79 Mo each)
- **2 versions** of externals (`externals.2.332.0` + `externals.2.333.0`, ~590 Mo each)

Total duplication: ~(79+590) x 2 versions x 5 runners = **~6.7 Go** of identical files.

---

## Actions

### 1. Delete old binary versions (bin.2.332.0 / externals.2.332.0)
Removed from all 5 runners that still had the previous version alongside the current one.

### 2. Delete root-level duplicates
`/opt/actions-runner/bin/` and `/opt/actions-runner/externals/` were standalone copies unused by any runner.

### 3. Replace copies with hardlinks (`cp -al`)
`bin/` and `externals/` in each secondary runner now hardlink to `copro-pilot/bin.2.333.0/` and `copro-pilot/externals.2.333.0/`. No additional disk space is consumed (refcount = 6, same inode).

> **Note:** Symlinks were tested first but rejected. The runner binary resolves the real path of the symlink and looks for `.runner`/`.credentials` in the source directory, making independent configuration impossible.

### 4. Re-register runners via GitHub CLI
Tokens generated automatically:
```bash
gh api -X POST repos/Wifsimster/<repo>/actions/runners/registration-token
```
No manual UI interaction required.

### 5. Reinstall systemd services
All 6 `actions.runner.Wifsimster-*.docker-server.service` services were cleanly reinstalled via `svc.sh`.

## Final Architecture

```
/opt/actions-runner/
├── copro-pilot/          # Binary source (1.4 Go real data)
│   ├── bin.2.333.0/      # Real files — inode refcount = 6
│   ├── externals.2.333.0/
│   ├── bin -> bin.2.333.0
│   └── externals -> externals.2.333.0
├── personal-blog/        # 0 extra space (hardlinks)
│   ├── bin/              ← hardlinks to copro-pilot/bin.2.333.0/*
│   └── externals/        ← hardlinks to copro-pilot/externals.2.333.0/*
├── toko/
├── resume/
├── wawptn/
└── x-ai-weekly-bot/
```

## Maintenance Notes

When a runner auto-updates to a new version (e.g. `bin.2.334.0`):

1. Recreate hardlinks for secondary runners:
```bash
for d in personal-blog toko resume wawptn x-ai-weekly-bot; do
  cp -al /opt/actions-runner/copro-pilot/bin.2.334.0 /opt/actions-runner/${d}/bin
  cp -al /opt/actions-runner/copro-pilot/externals.2.334.0 /opt/actions-runner/${d}/externals
done
```
2. Delete old version after stabilization:
```bash
for d in copro-pilot personal-blog toko resume wawptn x-ai-weekly-bot; do
  rm -rf /opt/actions-runner/${d}/bin.2.333.0
  rm -rf /opt/actions-runner/${d}/externals.2.333.0
done
```
